# 心理测试自动化使用说明

以后我想生成测试页时，不需要再说完整的 `test-page-automation`。

我只要说：

> 心理测试：你到底适合被爱还是主动爱？

或者：

> 生成心理测试，主题是：你是清醒型还是上头型

你就会自动按照 `test-page-automation` 这套流程走。

## 自动化流程

1. 根据我给的主题，生成适合公众号/小红书传播的测试标题。
2. 写测试开篇，让用户有代入感和点击欲。
3. 生成不少于 30 道测试题。
4. 每道题生成不同的 A/B/C/D 选项，不能所有题都用同一套选项。
5. 生成 4 个结果类型，每个结果要有情绪价值和可截图分享的话。
6. 根据主题选择视觉风格：
   - 情感、恋爱、女性向：偏柔和、粉色、奶油色、女性喜欢的风格。
   - 职业、赚钱、人生方向：偏中性、奶油色、灰绿色、清爽克制。
7. 生成独立测试链接，不覆盖以前的测试。
8. 发布前检查：
   - 是否有 30 道题。
   - 前两题选项是否重复。
   - 页面文件是否生成完整。
   - 链接路径是否稳定。
9. 如果 GitHub 发布失败，先诊断：
   - 授权问题。
   - 网络问题。
   - Token 权限问题。
   - 本地和 GitHub 版本冲突。
